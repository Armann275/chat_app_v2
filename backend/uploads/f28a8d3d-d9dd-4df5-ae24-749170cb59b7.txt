Award Manual Bonus
This method is used whenever the Casino Operator would like to award Bonus directly to specified Player without creating a tracked Promotion.

Authorizations:
ApiKeyAuth
 API Key: ApiKeyAuth
Header parameter name: X-Api-Token
Request Body schema: application/json
required
The settings needed for awarding a Bonus to a group of players.

id
required
string <uuid>
The idempotency key of the request. Must be a UUID represented by a standard 36-char string.

playerIds
required
Array of strings [ 1 .. 5000 ] items [ items [ 1 .. 64 ] characters ]
The identifiers of the players to be awarded this bonus. A maximum of 5 000 (five thousand) players may be awarded per request. If the total number of players exceeds this limit, split the request into multiple batches.

skinName
required
string [ 1 .. 128 ] characters
The name of the players' skin.

amount	
integer <int64> > 0
The quantity of bonuses being awarded per player. Constraints: The parameter is required for all award types except COUNTDOWN_FREE_SPIN, for which it may be omitted or set to null. The parameter must be set to 1 for FREE_BUY_BONUS.

type
required
string
Enum: "FREE_SPIN" "FREE_BUY_BONUS" "GOLDEN_CHIP" "COUNTDOWN_FREE_SPIN"
The type of the bonuses being awarded.

typeConfiguration	
object
Any additional configuration details needed for the specified bonus type.

freeBuyBonusConfiguration	
object
Configuration for "Free Buy Bonus" bonuses, defining the awarded feature. Constraint: The parameter is required for FREE_BUY_BONUS

goldenChipConfiguration	
object
Configuration for "Golden Chip" bonuses, defining the betting restrictions. Constraint: The parameter is required for GOLDEN_CHIP.

countdownFreeSpinConfiguration	
object
Configuration for "Countdown Free Spin" bonuses. Constraint: The parameter is required for COUNTDOWN_FREE_SPIN

betValueStrategy
required
string
Enum: "MINIMUM" "EXACT" "CLOSEST" "CLOSEST_LOWER" "CLOSEST_HIGHER"
The strategy used for determining the bet value.

betValue	
number > 0
The value of the bonuses awarded. The parameter may be omitted if the Bet Value Strategy used is MINIMUM. For any other Bet Value Strategy, the parameter is required.

currency
required
string = 3 characters
The currency code for the bonuses, defined by the three-letter currency code as per ISO-4217.

games
required
Array of integers <int32> [ 1 .. 1000 ] items [ items <int32 > > 0 ]
The unique identifiers of the games eligible for the bonuses.

claimByStrategy	
string
Enum: "RELATIVE_TO_AWARD" "CUSTOM"
The strategy for determining the deadline by which the bonuses must be claimed.

claimByMinutes	
integer <int64> [ 0 .. 1440000 ]
The number of minutes the bonuses are available to be claimed after being awarded. Required if Award Manual Bonuses Claim By Strategy is RELATIVE_TO_AWARD.

claimByDateTime	
string <date-time>
The deadline by which bonuses must be claimed. (UTC time zone)

expirationStrategy	
string
Enum: "RELATIVE_TO_CLAIM" "CUSTOM"
The strategy for determining when the bonuses will expire.

expirationMinutes	
integer <int64> ( 0 .. 1440000 ]
The number of minutes the bonus is valid for after being claimed. Required if expirationStrategy is RELATIVE_TO_CLAIM.

expirationDateTime	
string <date-time>
The exact date when the bonus will expire. Required if expirationStrategy is CUSTOM.

Responses
200 Bonuses awarded successfully.
